Staff Management System

Project Overview:

This Software as a Service system was designed to manage the scheduling, tracking, and reporting of officer duties at various sites. The solution encompassed multiple complex features aimed at improving efficiency and automating manual processes.

Key Features:

  • Real-Time Scheduling:: Automated scheduling of officers for various sites, ensuring that duties were assigned efficiently and in real-time.
  • Notifications: Officers and site managers received notifications regarding duty schedules, check-ins, check-outs, and leave approvals.
  • Reports Generation: Generated detailed reports on officer duties, attendance, and overall site performance.
  • Messaging: Included messaging features for seamless communication between officers and site managers.
  • Officer Pre-check-in/Check-in/Check-out (Mobile App): Handled backend logic for officer check-in and check-out, integrated with a mobile app.
  • Leave Management System: Leave requests, approval workflows, and tracking.
  • Screening and Vetting for Signup: Introduced a sign-up process that required thorough vetting and approval before officers could join.

User Authentication and Permissions:

This system was built around a role-based access control (RBAC) mechanism. Different users, such as officers, site managers, and admins, had specific roles with varying levels of access to features. Authentication was handled via token-based security, ensuring secure access to the system, while permissions were tightly managed according to user roles.

Challenges and Data Synchronization:

Managing real-time data and synchronizing across the mobile app and web interface posed a challenge. However, the use of caching (e.g., Redis) and optimized database queries ensured that all actions—scheduling, notifications, and reports—were reflected instantly across the system. Additionally, managing heavy traffic during peak check-in/out times was optimized using load balancing.

Distributed System Architecture

Project Overview:

This project focused on addressing the scalability challenges of a Django-based application serving over 10,000 users. The goal was to enhance the architecture to allow seamless growth without compromising performance.

Problem Solved:

The core issue was scalability. The existing infrastructure could not efficiently handle the growing number of users, leading to performance bottlenecks.

Technologies and Approach:

  • Django: The core framework for the project.
  • Docker Containers: Used to separate services for modularity and easy scalability.
  • Azure VMSS (Virtual Machine Scale Sets): Enabled dynamic scaling of virtual machines based on traffic demands.
  • Azure Database for MySQL and MongoDB:: MySQL was deployed on Azure, while MongoDB required a separate server since Azure did not offer out-of-the-box support for it.
  • Memcache: Deployed on a separate server for caching to improve response times.
These improvements allowed the system to handle a larger load efficiently while optimizing costs through dynamic scaling.

Improvement to Existing Architecture:

By transitioning to a distributed architecture, the project gained the following benefits:

  • Improved Scalability: The use of VMSS and Docker containers allowed the system to scale dynamically as user traffic increased, ensuring high availability and performance.
  • Service Separation: Separating services like MySQL, MongoDB, and Memcache ensured that these resources were optimized and dedicated to specific tasks, reducing the load on individual instances.
  • Cost Efficiency: Dynamic scaling reduced operational costs by spinning up VMs only when needed, thereby optimizing resource utilization.
These improvements allowed the system to handle a larger load efficiently while optimizing costs through dynamic scaling.

Open edX Customization

Project Overview:

This project involved customizing Open edX to transform it into a Software-as-a-Service (SaaS) platform, allowing multiple organizations to have their own uniquely branded learning platforms. Additional functionalities, like OTP authentication, were also integrated.

Key Customizations:

  • SaaS Model: The core LMS was modified to support a multi-tenant structure, enabling different organizations to sign up and have exclusive themes. Each organization could customize their learning environment, creating a unique experience for their users.
  • OTP on User Login: Added an extra layer of security by implementing an OTP (One-Time Password) functionality during login, improving user security.
  • Custom Themes: Organizations were able to customize the look and feel of their LMS, resulting in a more personalized user experience.

Impact on User Experience:

  • Improved Personalization: The introduction of custom themes allowed each organization to offer a tailored experience for their users, aligning the platform’s design with their branding.
  • Enhanced Security: The OTP feature significantly improved login security, which was particularly beneficial for institutions handling sensitive data.
  • Ease of Access: The SaaS model made it easier for organizations to onboard and deploy their own LMS without needing to manage the backend infrastructure, making the platform more scalable and user-friendly.
