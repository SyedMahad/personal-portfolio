Distributed System Architecture

Project Overview:

This project focused on addressing the scalability challenges of a Django-based application serving over 10,000 users. The goal was to enhance the architecture to allow seamless growth without compromising performance.

Problem Solved:

The core issue was scalability. The existing infrastructure could not efficiently handle the growing number of users, leading to performance bottlenecks.

Technologies and Approach:

  • Django: The core framework for the project.
  • Docker Containers: Used to separate services for modularity and easy scalability.
  • Azure VMSS (Virtual Machine Scale Sets): Enabled dynamic scaling of virtual machines based on traffic demands.
  • Azure Database for MySQL and MongoDB:: MySQL was deployed on Azure, while MongoDB required a separate server since Azure did not offer out-of-the-box support for it.
  • Memcache: Deployed on a separate server for caching to improve response times.
These improvements allowed the system to handle a larger load efficiently while optimizing costs through dynamic scaling.

Improvement to Existing Architecture:

By transitioning to a distributed architecture, the project gained the following benefits:

  • Improved Scalability: The use of VMSS and Docker containers allowed the system to scale dynamically as user traffic increased, ensuring high availability and performance.
  • Service Separation: Separating services like MySQL, MongoDB, and Memcache ensured that these resources were optimized and dedicated to specific tasks, reducing the load on individual instances.
  • Cost Efficiency: Dynamic scaling reduced operational costs by spinning up VMs only when needed, thereby optimizing resource utilization.
These improvements allowed the system to handle a larger load efficiently while optimizing costs through dynamic scaling.
